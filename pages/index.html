_start — Ouvrir le socket

« On commence par _start. On cree un socket TCP avec SYS_SOCKET, on active SO_REUSEADDR pour pouvoir relancer le serveur, puis bind sur le port 8080. »

  • 0x901F = htons(8080) en network byte order
  • r12 = fd du socket serveur
  • listen avec file d'attente de 10
Noah — Reseau
http_server.asm — ligne 577
580_start:
581    mov     rax, SYS_SOCKET
582    mov     rdi, AF_INET
583    mov     rsi, SOCK_STREAM
584    xor     rdx, rdx
585    syscall
587    mov     r12, rax       ; fd socket
597    mov     rax, SYS_BIND
599    mov     rsi, srv_addr  ; port 8080
605    mov     rax, SYS_LISTEN
607    mov     rsi, 10

.loop — Accepter les clients

« Ensuite une boucle infinie : accept bloque jusqu'a une connexion, on lit la requete dans req_buf, on appelle notre code HTTP, puis on ferme. »

  • r13 = fd du client connecte
  • req_buf = 4096 octets de requete brute
  • Puis call parse_request
Noah — Reseau

637    call    parse_request

663    mov     rax, SYS_CLOSE  ; ferme client

parse_request — Lire le GET

« Ici on compare le debut de req_buf avec "GET " ou "POST ". Puis on copie le chemin caractere par caractere dans path_buf jusqu'a l'espace. »

  • Entree : req_buf = GET /about HTTP/1.1…
  • Sortie : path_buf = "/about"
  • Si methode inconnue → rax = -1 → 405

564    mov     byte [rdi], 0  ; null-terminate

route_request — Choisir la page

« On compare path_buf avec nos routes connues via str_eq. Si ca match, on ouvre le fichier HTML sur le disque. Sinon → send_404. »

  • / → pages/index.html (cette presentation)
  • /status → JSON genere en memoire
  • /style.css → CSS avec Content-Type adapte

send_body — Reponse HTTP

« On envoie d'abord le header pre-construit en .data, puis le corps HTML. Les \r\n sont les octets 13 et 10 — obligatoires en HTTP. »

  • 2 syscalls write sur le fd client (r13)
  • Header 200, 404, CSS ou JSON selon r15b
  • C'est ce qui s'execute maintenant pour afficher cette page
Mathéo — HTTP
http_server.asm — .data + send_body
100    resp_200_hdr:
101        db  "HTTP/1.1 200 OK", 13, 10
102        db  "Content-Type: text/html", 13, 10
104        db  13, 10              ; ligne vide = fin headers
233send_body:
240    mov     rsi, resp_200_hdr
255    mov     rax, SYS_WRITE  ; 1. header
256    mov     rdi, r13        ; fd client
259    mov     rax, SYS_WRITE  ; 2. corps HTML
261    mov     rsi, file_buf
262    mov     rdx, r14        ; taille lue

Bonus — le code en plus

« Au-dela du sujet, on a ajoute les logs dans le terminal, une 404 personnalisee, le POST, et plusieurs types MIME pour servir du CSS et du JSON. »

  • log_request → affiche [LOG] GET / a chaque requete
  • send_404 → sert pages/404.html depuis le disque
  • read_post_body → lit le corps POST via Content-Length
  • r15b choisit le header : HTML, CSS ou JSON
✓ Routes multiples
✓ Fichiers disque
✓ Logs terminal
